  1. HTTP method names must be tokens

1. HTTP method names must be tokens

问题描述

突然的一天收到这样的一条信息，“服务端对应的各个客户端（小程序，IOS，Android，PC端）用户都无法正常操作了”。然后获得服务器的日志，抛出的异常信息如下，‘Invalid character found in method name. HTTP method names must be tokens’。

分析过程

  1. 使用restClient 测试随便一条请求，分别使用 “http:// + ip”, “https:// + ip” , “https:// + 域名” ， “https:// + ip” 的方式。
  2. 发现 “http:// + ip” 的方式可以正常请求，其他则不行，其中 “https:// + ip” 的时候，谷歌浏览器中的 restClient 提示信息“No response. Is the certificate valid? Click here to check.”，点击之后打开的新的页面提示：“隐私设置错误”
  3. 从网上百度获得一条信息：“本来是http的请求 错误使用https方式请求 就会报该错”

综合分析，ssl 的证书过期了

解决办法

到https证书签发机构CA续签证书。
